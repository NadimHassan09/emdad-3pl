Use this backend folder structure inside /server/src:

src/
  common/
    constants/
    decorators/
    dto/
    enums/
    exceptions/
    guards/
    interceptors/
    interfaces/
    pipes/
    utils/
  config/
  database/
    prisma/
  modules/
    auth/
    clients/
    users/
    actors/
    warehouses/
    locations/
    uom/
    products/
    batches/
    inbound-orders/
    outbound-orders/
    stock-reservations/
    inventory/
    task-work-orders/
    adjustments/
    return-orders/
    approvals/
    billing/
    vas/
    audit/
  app.module.ts

For each module, prefer:
- controller
- service
- dto
- enums
- types/interfaces if needed
- mapper if needed
- module file

Do not create unnecessary abstraction layers unless the module genuinely benefits from them.

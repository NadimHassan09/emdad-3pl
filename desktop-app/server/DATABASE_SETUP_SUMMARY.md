## 🚀 Quick Start

### Step 1: Set PostgreSQL Admin Credentials (if different from defaults)

**Default credentials assumed:**
- Username: `postgres`
- Password: `postgres`

**If your PostgreSQL uses different admin credentials, set them:**

**Windows PowerShell:**
```powershell
$env:POSTGRES_ADMIN_USER="your_admin_user"
$env:POSTGRES_ADMIN_PASSWORD="your_admin_password"
```

**Windows CMD:**
```cmd
set POSTGRES_ADMIN_USER=your_admin_user
set POSTGRES_ADMIN_PASSWORD=your_admin_password
```

### Step 2: Run Setup

```bash
cd server
npm run db:setup
```

This will:
1. ✅ Create database user `emdad_3pl_app` (if missing)
2. ✅ Create database `emdad_3pl` (if missing)
3. ✅ Grant all privileges to the app user
4. ✅ Run `schema.sql` to create all tables, enums, triggers
5. ✅ Update `.env` file with generated credentials

### Step 3: Generate Prisma Client

```bash
npm run prisma:generate
```

### Step 4: (Optional) Seed with Fake Data

## 🔌 How Backend Connects

The NestJS backend reads database configuration from:

1. **Environment Files**: `.env.local` (preferred) or `.env` (fallback)
2. **Variable**: `DATABASE_URL` (Prisma connection string)
3. **Format**: `postgresql://USER:PASSWORD@HOST:PORT/DATABASE?schema=public`

**Connection Flow:**
```
.env/.env.local 
  → DATABASE_URL 
  → PrismaClient (in prisma/schema.prisma)
  → PrismaService 
  → Application Code
```
